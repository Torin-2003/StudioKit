# StudioKit Full i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Translate all main operation areas (titles, buttons, status messages, labels) in `hypecutter/ui.py` and `scene_manager/ui.py` into EN/ZH using the existing `t()` system, skipping technical config labels (API Key, Model, Base URL, etc.).

**Architecture:** Extend `i18n.py` with ~80 new translation keys for HypeCutter and ~90 for Scene Manager. Both `ui.py` files import `t` from `i18n` and replace hardcoded English strings with `t(key, lang)` calls. The `lang` value is read from `st.session_state.get("lang", "en")` at the top of each `render()` function.

**Tech Stack:** Python 3.11, Streamlit ≥1.35, existing `i18n.py` module

**Key constraint:** Technical terms stay English: "API Key", "Base URL", "Model", "Whisper", "GPT", "OpenAI", "Anthropic", "Gemini", resolution values ("720p"), model names ("base", "small"), provider names. These are industry-standard terms Chinese users also use in English.
